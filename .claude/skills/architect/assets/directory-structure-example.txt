Key Principles:
===============

1. Process Separation
   - main/ contains Node.js code
   - renderer/ contains browser code
   - preload/ bridges the two securely
   - shared/ for common code/types

2. Feature-Based Organization
   - Group related files together
   - Makes features easy to find and modify
   - Facilitates code splitting

3. Clear Boundaries
   - Services handle business logic
   - Repositories handle data access
   - Components handle UI
   - IPC handlers coordinate processes

4. Type Safety
   - Shared types ensure consistency
   - Preload types provide autocomplete
   - Separate configs for each process

5. Testability
   - Organized tests mirror source structure
   - Unit, integration, and E2E tests separated
   - Easy to mock services and dependencies
